# 数据库概览

本项目（Litho/deepwiki-rs）不使用传统数据库。系统的持久化需求通过以下方式满足：

- **缓存存储**：LLM 调用结果缓存使用 JSON 文件存储在 `.litho/cache/` 目录下，由 CacheManager 管理
- **外部知识缓存**：PDF/MD/SQL 等外部文档的处理结果缓存存储在 `.litho/cache/knowledge/` 目录下
- **Memory 数据**：使用 HashMap<String, Value> 的内存存储，仅在单次运行周期内有效，不持久化
- **文档输出**：最终生成的文档以 Markdown 文件形式持久化到输出目录

Litho 本身是一个 CLI 工具，不是长期运行的服务，因此不需要数据库来维护状态。所有需要跨运行周期持久化的数据（缓存）都使用文件系统存储。
