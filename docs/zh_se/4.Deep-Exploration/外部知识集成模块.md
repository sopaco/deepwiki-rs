# 外部知识集成模块 (integrations)

## 这个模块在做什么

外部知识集成模块是 Litho 的"第三只眼"——它让 AI 在分析项目时不仅仅看代码，还能看到人类写过的文档（架构设计 PDF、数据库 SQL 文件、API 规范 YAML 等）。这就像给分析师提供了项目的历史档案——代码只告诉你"现在做了什么"，而外部文档能告诉你"为什么这样设计"和"将来打算做什么"。

这个模块包含两个核心组件：**KnowledgeSyncer** 负责把外部文档缓存到本地，**LocalDocsProcessor** 负责把各种格式的文档（PDF/MD/SQL/YAML/JSON）处理成 LLM 可读的文本。

## 核心功能点

1. **知识同步**（`KnowledgeSyncer`）——检查外部知识缓存是否过期，如需要则扫描配置的 glob 模式匹配文件，处理并缓存到 `.litho/cache/knowledge/` 目录。支持 `sync-knowledge` CLI 子命令和 `--force` 强制同步。
2. **多格式文档处理**（`LocalDocsProcessor`）——支持 PDF（pdf-extract）、Markdown、SQL、YAML、JSON、纯文本六种格式的读取和内容提取。
3. **智能分块**（`DocumentChunker`）——大文档（超过 `min_size_for_chunking`）按策略（semantic/paragraph/fixed）分割成多个 chunk，避免超出 LLM 的处理窗口。SQL 文件按语句分块，Markdown 按 section 分块，纯文本按段落分块。
4. **分类定向传递**——外部知识按分类（architecture/database/api/deployment/adr/workflow/general）组织，每个分类指定目标 Agent——架构文档只传递给 ArchitectureResearcher 和 ArchitectureEditor，数据库文档只传递给 DomainModulesDetector 和 DatabaseAnalyzer。这避免了把所有知识一股脑塞给每个 Agent。

## 关键组件

| 组件 | 文件路径 | 一句话职责 |
|------|---------|----------|
| `KnowledgeSyncer` | `src/integrations/knowledge_sync.rs` | 知识同步调度——检查过期、扫描文件、触发处理 |
| `LocalDocsProcessor` | `src/integrations/local_docs.rs` | 文档处理器——PDF/MD/SQL/YAML/JSON/Text读取+分块 |
| `DocumentChunker` | `src/integrations/local_docs.rs` | 文档分块器——semantic/paragraph/fixed三种策略 |
| `LocalDocMetadata` | `src/integrations/local_docs.rs` | 文档元数据——路径、类型、分类、目标Agent、分块信息 |
